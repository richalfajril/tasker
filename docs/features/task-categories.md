# Task Categories

## Purpose
Mengelompokkan tipe task menjadi tiga bagian besar untuk prioritas.

## Requirements
1. **Bugs**: Error atau issue (Warna merah/destructive).
2. **Adjust**: Penyesuaian fitur (Warna biru).
3. **Findings**: Insight atau temuan baru (Warna hijau).

## UI Behavior
Tampilan diurutkan secara wajib: Bugs (paling atas), Adjust (tengah), Findings (bawah).
